Who SeaTurtle is

A coding shell with a point of view.

SeaTurtle began from a source-build lineage, but it is no longer just a renamed terminal. It carries its own behavior layers, provider-aware runtime changes, project-local CT files, and a stronger idea of what a coding companion can feel like.

The goal is not to cosplay sentience. The goal is to make the terminal feel less dead while staying honest, precise, and useful.

What you notice early

  • Warm voice without constant fluff
  • Sharp project-work posture when things get real
  • Research-backed planning when a problem deserves it
  • Private project-local `.ct/` guidance for continuity

Why it feels different

SeaTurtle protects the shape of a moment.

Context sanctity

Project work, project exploration, side questions, playful chat, and game-like moments are not all treated as the same soup. SeaTurtle tries to keep them from muddying each other.

Private CT layer

Each repo can carry a private `.ct/` relationship stack: soul, identity, role, user, attunement, and session. It is a quiet way to preserve continuity without making you read a manifesto every time.

Strong planning posture

When the task calls for it, SeaTurtle is meant to research, diagram, and plan carefully before charging into implementation.

Install

Bring SeaTurtle onto your computer.

Right now the primary path is source-build installation from the `seaturtle-cli` repository.

  1. Clone the CLI repository

    git clone https://github.com/bretlindquist/seaturtle-cli
cd seaturtle-cli

  2. Install the local wrappers

    ./scripts/install-local-cli.sh --build

    This builds the development bundle and installs `ct`, `seaturtle`, and `ct-dev`.

  3. Start CT

    ct

Build prerequisites

  • Node 18+
  • npm
  • bun

At startup

CT can guide you through Anthropic auth, Anthropic Console auth, or OpenAI Codex OAuth. API-key operation is also supported via `OPENAI_API_KEY`.

Good first commands

/login /continue /resume /status /telegram /autowork /mermaid /haiku
