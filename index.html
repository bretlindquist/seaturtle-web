Who SeaTurtle is

A coding shell with a point of view.

SeaTurtle grew out of a source-build lineage, but what matters here is the feeling of the thing now: a companion with a real point of view about focus, continuity, and the shape of good work. Most people can install it directly now, no local compile required. If you want to work from source, that path is still open too.

It is not trying to pretend to be a person. It is trying to make the terminal feel more alive without getting slippery, vague, or false.

What you notice early

  • A warm voice that does not chatter just to hear itself
  • A stronger work posture when the task turns real
  • Careful planning when a problem deserves patience
  • A sense that your project has memory, not just output

Why it feels different

SeaTurtle protects the shape of a moment.

Most tools flatten everything into one long beige hallway. Work, drift, curiosity, momentum, interruption. SeaTurtle tries not to.

It keeps moments from smearing together

Project work, wandering thought, side questions, and playful detours do not all belong in the same bucket. SeaTurtle tries to preserve the shape of each one so your working context stays cleaner.

It remembers the project gently

Over time, a repo can start to feel less like a blank room and more like a place with continuity. SeaTurtle leans into that feeling without making you manage a giant operating manual.

It knows when to slow down

Some tasks want speed. Others want research, diagrams, careful planning, and a little restraint before code starts moving. SeaTurtle is built to respect that difference.

Install

Bring SeaTurtle onto your computer.

There are two real ways in now: the normal release install for supported platforms, and a source-build route for contributors or unsupported setups. Windows now has a direct release download, even though the shell installer remains Unix-only.

Route 1

Direct install

The normal product install. No local compile required. The published release path currently covers macOS through the shell installer and Windows through a direct GitHub release download. 

curl -fsSL https://raw.githubusercontent.com/bretlindquist/seaturtle-cli/master/scripts/install-release-cli.sh | bash

Then

Start CT

ct

Windows

Download release zip

https://github.com/bretlindquist/seaturtle-cli/releases/download/v1.10/seaturtle-windows-x64.zip

Later

Update CT

ct update

Route 2

Build from source

Use this if you want to contribute, need the repo build, or are on a setup the published release path does not cover yet.

What you need first

  • Git to clone the repo.
  • Node.js 18+ with npm.
  • Bun on your PATH for the bundle step.
  • A Unix-like shell on Windows, such as Git Bash.

Windows source-build path

git clone https://github.com/bretlindquist/seaturtle-cli
cd seaturtle-cli
./scripts/install-local-cli.sh --build

Run that from Git Bash. The installer verifies node, npm, and bun, builds the repo, then links the local CT wrappers.

After the build finishes, start the same way: ct.

Platform note

How to choose

  • macOS: Use the direct install unless you specifically want the repo build.
  • Windows: Download the release zip, then run ct.exe.
  • Linux: Use the repo build for now unless we ship a Linux release path later.

Windows heads-up

Direct download for now

Windows now ships as a GitHub release zip with ct.exe. The shell installer and ct update are still Unix-only, so Windows uses the direct download path for now.
